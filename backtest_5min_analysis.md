## 六、优化改进方案

### 6.1 参数层面优化

#### 动量策略

| 参数 | 当前默认值 | 建议值 | 理由 |
|------|-----------|--------|------|
| `spread_entry` | 0.03% | **0.01%** | 5分钟横盘期EMA张口极小，需降低门槛 |
| `slope_entry` | 0.015% | **0.005%** | 斜率阈值过高导致信号完全沉默 |
| `short_p` | 5 | **3** | 缩短EMA响应时间，捕捉更微观的动量 |
| `long_p` | 15 | **10~15** | 保持快慢线适度分离 |
| `cooldown_bars` | 2 | **0~1** | 短周期内冷却期会过滤掉大部分机会 |

#### 反转策略

| 参数 | 当前默认值 | 建议值 | 理由 |
|------|-----------|--------|------|
| `rsi_period` | 5 | **8** | 5周期RSI噪声过大，8周期更稳定 |
| `rsi_oversold` | 28 | **32~35** | 放宽超卖门槛，增加信号频率 |
| `rsi_overbought` | 72 | **65~68** | 对称放宽 |
| `deviation_entry` | 0.15% | **0.10%** | 降低EMA偏离度门槛 |
| `min_score` | 0.35 | **0.30** | 降低综合评分门槛，提高开仓频率 |
| `cooldown_bars` | 1 | **0** | 短周期反转机会稍纵即逝 |

### 6.2 架构层面优化

#### 1. 自适应参数切换（最重要）

当前静态参数无法适应不同的市场状态。建议引入**市场状态识别器**：

```
IF ATR(20) > 阈值A → 高波动状态 → 使用动量策略（趋势跟踪）
IF ATR(20) < 阈值B → 低波动状态 → 使用反转策略（均值回归）
IF 阈值B ≤ ATR ≤ 阈值A → 中性状态 → 降低仓位或暂停交易
```

在本例中，5分钟ATR极低，系统应自动切换为「低波动反转模式」或停止交易。

#### 2. 波动率过滤（Volatility Filter）

为策略添加最小波动率门槛：
- 若最近N根bar的CV（变异系数）< 0.01%，策略输出forced_neutral
- 这能避免在"死市"中频繁交易消耗成本

#### 3. 成本敏感性检查

在回测引擎中增加**盈亏平衡分析**：
- 计算每笔交易的预期收益 vs 固定成本（手续费+滑点）
- 若 `|预期收益| < 2×成本`，标记为"成本敏感交易"
- 本例中反转策略平均单笔收益仅0.0019%，而成本约0.07%/2=0.035%，**所有交易都是成本敏感型**

#### 4. 多时间框架确认（MTF）

5分钟tick回测过于微观，建议：
- 用1分钟或5分钟K线计算"大局方向"
- 若大局方向为下跌，反转策略只做short（反弹做空），不做long
- 本例中5分钟大局微跌，反转Long-Short的13笔交易中，做空盈利可能贡献了主要正收益

#### 5. 信号置信度加权仓位

当前策略是"全仓/空仓"二元切换。建议改为：
- `position_size = f(signal_strength, volatility, trend_alignment)`
- 例如：strength=30时开30%仓位，strength=90时开90%仓位
- 在弱信号时降低仓位，减少随机成本损耗

### 6.3 策略组合优化

本例中动量+反转的信号一致性高达81.1%（同时neutral），说明两策略在低波动期都缺乏信心。建议：

1. **策略开关机制**：
   - 动量信号为neutral且反转信号为neutral时 → **空仓观望**
   - 仅当一个策略产生strong信号时 → **开仓**
   - 两策略同时发出反向信号时 → **优先反转策略**（因反转在短周期更稳健）

2. **组合打分**：
   ```
   combined_score = momentum_strength × 0.4 + reversal_score × 0.6
   ```
   反转策略权重更高，因其在短周期统计上更适应日内噪声。

## 七、风险提示与局限性

1. **样本量极小**：5分钟、159根1s bar，任何"统计结论"都不具备显著性
2. **过拟合风险**：网格搜索在单窗口上的最优参数很可能只是噪声拟合
3. **成本吞噬**：在0.07%的交易成本下，5分钟内需要至少0.1%的波动空间才能盈利
4. **数据质量**：tick数据来自Infoway WebSocket推送，可能存在丢包或延迟
5. **单一窗口**：建议至少回测30~50个不重叠的5分钟窗口，才能评估策略稳健性

**核心建议**：
1. **不要在这个5分钟窗口交易** — 波动率太低，无法覆盖成本。
2. **必须引入波动率过滤器** — 低于阈值时强制空仓。
3. **动量策略需大幅降低参数门槛** 才能在高频场景下有用。
4. **反转策略参数有较大优化空间**（rsi_period=8, deviation=0.1%等）。
5. **回测应扩展到50+个5分钟窗口** 再做参数优化，避免过拟合。
